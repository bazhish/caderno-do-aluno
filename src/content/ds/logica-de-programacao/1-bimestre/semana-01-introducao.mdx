## Aula Teórica Completa

Antes de aprender qualquer linguagem de programação (Python, JavaScript, C, etc.), é preciso entender a lógica que existe **por trás** do código. Essa lógica tem dois pilares: o **algoritmo** e a **variável**.

### O que é um algoritmo

Um **algoritmo** é uma sequência lógica, finita e ordenada de passos para resolver um problema específico. Não precisa envolver computador nenhum: uma receita de bolo é um algoritmo (passos ordenados para chegar a um bolo pronto), assim como um manual de montagem de móvel.

Todo algoritmo de computador segue, de forma bem geral, três grandes etapas:

1. **Entrada** — os dados que o programa recebe (ex: o usuário digita duas notas).
2. **Processamento** — o que o programa faz com esses dados (ex: soma as notas e divide por dois).
3. **Saída** — o resultado que o programa entrega (ex: mostra a média na tela).

Essa ordem — Entrada → Processamento → Saída — é a espinha dorsal de praticamente todo programa que existe, do mais simples ao mais complexo.

### O que é uma variável

Uma **variável** é um espaço nomeado guardado na memória do computador, usado para armazenar um valor que pode mudar ao longo da execução do programa. Pense nela como uma caixa com etiqueta: a etiqueta é o **nome** da variável (por exemplo, `nota1`), e dentro da caixa fica o **valor** guardado naquele momento (por exemplo, `8.5`).

**Exemplo de atribuição:** nota1 = 8.5
- **nota1** é o nome da variável — a "etiqueta da caixa".
- **=** é o operador de atribuição — ele não significa "igual" como na matemática, significa "guarde o valor da direita dentro da variável da esquerda".
- **8.5** é o valor que está sendo guardado dentro da variável.

O valor guardado em uma variável pode ser trocado depois: se, mais adiante no programa, você escrever `nota1 = 9.0`, o conteúdo da caixa muda de 8.5 para 9.0 — por isso o nome "variável", ao contrário de uma **constante**, que guarda um valor fixo que nunca muda durante a execução.

**Algoritmo** é uma sequência lógica e finita de passos para resolver um problema — não depende de nenhuma linguagem de programação específica, é a ideia antes do código.

Todo algoritmo segue, de forma geral, a estrutura **Entrada → Processamento → Saída**: primeiro os dados entram, depois são transformados, por último o resultado é entregue.

**Variável** é um espaço nomeado na memória que guarda um valor que pode mudar durante a execução do programa — pense nela como uma caixa com etiqueta (o nome) e um conteúdo (o valor).

O operador **"="** em programação não significa igualdade matemática, significa **atribuição**: "guarde o valor da direita na variável da esquerda".

**Constante** é o oposto de variável: um valor fixo que não muda ao longo da execução do programa.
